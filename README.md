# GeoSat-Algebra: Geometric and Conformal Algebra for Satellite Geometry and Pose Control

## üöÄ Overview

This repository explores the application of **Geometric Algebra (GA)** and its specialization, **Conformal Geometric Algebra (CGA)**, as a powerful and unified mathematical framework for solving complex 3D geometry and transformation problems inherent in satellite systems and aerospace engineering.

By leveraging GA/CGA, we aim to demonstrate mathematically robust and computationally efficient alternatives to traditional methods (matrices, quaternions) for tasks such as:
1.  **Attitude Determination and Control Systems (ADCS):** Using Rotors (versors) for singularity-free attitude control.
2.  **On-Orbit Collision Avoidance:** Efficient geometric calculation of distances and intersections using CGA blades.

## üéØ Core Focus: CGA in 5D Space

The project focuses on implementing algorithms within $G(4,1)$‚Äîthe 5D Conformal Geometric Algebra‚Äîwhich integrates translation, rotation, and scaling into a single algebraic structure, essential for dynamic optimization in satellite constellations.

## üí° Key Advantages of GA/CGA

| Feature | Traditional Methods | Geometric Algebra (GA/CGA) |
| :--- | :--- | :--- |
| **Unified Representation** | Requires separate data types for geometry (points, lines, planes) and algebra (vectors, matrices). | All geometric objects and transformations are unified as **Multivectors**. |
| **Transformation** | Separate operations for rotation (quaternions/matrices) and translation (vector addition). | All transformations are performed using the **Geometric Product** (versor multiplication), simplifying command sequences. |
| **Robustness** | Avoids issues like Gimbal Lock, crucial for real-time control systems. |

## üèÉ Running the Demos

To run any demonstration, activate the environment and execute the file. Results (e.g., plots comparing a GA rotation vs. Matrix rotation) will be saved in `5_Results_Analysis/`.

### 1. GA Fundamentals (3D Rotation)

Demonstrates the Geometric Product and the use of Rotors in 3D Euclidean space ($G(3)$).

```bash
python src/demo_ga_3d_rotation.py
```

### 2. CGA Primitives (5D Objects)

Shows how Points, Lines, Planes, and Spheres are defined as multivectors in $G(4,1)$ and how they interact.

```bash
python src/demo_cga_primitives.py
```

### 3. Application: Collision Avoidance Geometry

A practical example demonstrating how CGA can calculate the closest distance between two spheres (satellites) or a sphere and a line (potential debris trajectory) using the meet and join operations.

```bash
python src/application_collision_avoidance.py
```
